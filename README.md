# Zoom Clone

A functional Zoom web app clone with a Next.js (App Router) frontend and a Python FastAPI + SQLite backend. Users can start instant meetings, join by ID, schedule meetings, and participate in a meeting room with host controls (mute all, remove participant) тАФ without any real video streaming.

Deployed as a **single Vercel project**: Next.js is served from the edge and the FastAPI backend runs as a Python serverless function under `/api/*`.

## Tech stack

| Layer     | Choice |
|-----------|--------|
| Frontend  | Next.js 16.2 (App Router, Turbopack, React Compiler), React 19.2, Tailwind CSS v4, lucide-react, date-fns |
| Backend   | Python 3.11+, FastAPI, SQLModel (SQLAlchemy), Pydantic v2 |
| Database  | SQLite тАФ `frontend/zoom_clone.db` locally, `/tmp/zoom_clone.db` on Vercel |
| Tooling   | Bun package manager, Biome (lint/format) |
| Hosting   | Vercel (Next.js + Python runtimes in one project) |

## Deploying to Vercel

The whole app deploys as **one Vercel project**. Vercel's Python runtime auto-detects `frontend/api/index.py` (ASGI export `app`), and the `vercel.json` rewrite forwards every `/api/*` request to that single FastAPI function.

### 1. Push the repo to GitHub/GitLab/Bitbucket

### 2. Import into Vercel

- **Framework Preset**: Next.js (auto-detected)
- **Root Directory**: `frontend` тАФ this is required
- **Build Command**: leave as default (`next build`)
- **Install Command**: leave as default (`bun install` / `npm install`)

Vercel picks up `frontend/requirements.txt` automatically and provisions the Python runtime for the function at `api/index.py`.

### 3. Environment variables (optional)

| Variable                   | Purpose                                                                 |
|----------------------------|-------------------------------------------------------------------------|
| `NEXT_PUBLIC_API_URL`      | Override the client-side API base. Leave **unset** in production тАФ the client defaults to same-origin so `/api/*` works via the rewrite. |
| `FRONTEND_URL`             | Overrides invite/join links generated by the API. Defaults to `VERCEL_PROJECT_PRODUCTION_URL` / `VERCEL_URL` when unset on Vercel. |
| `ZOOM_CLONE_DB_PATH`       | Optional SQLite path override (default `/tmp/zoom_clone.db` on Vercel). |

### 4. First request

On each cold start the function calls `seed_if_empty()`, which creates tables and seeds sample users/meetings only when the DB has zero users. Because Vercel's filesystem is read-only outside `/tmp`, the DB lives at `/tmp/zoom_clone.db` and does **not persist across cold starts** тАФ perfect for the RL-gym use case, not intended as a real datastore.

If you need durable data, swap the SQLite engine URL in `frontend/api/database.py` for a hosted DB (Turso / libSQL, Neon Postgres, PlanetScale, etc.). All ORM code is standard SQLModel and will keep working.

### 5. Deploy via CLI (alternative)

```bash
npm i -g vercel
cd frontend
vercel               # first time тАФ links project, remembers Root Directory
vercel --prod        # ship to production
```

## Local development

Two terminals. Both from the repo root.

### Terminal 1 тАФ API

Two options; pick one.

**Option A (recommended): use the Vercel-shaped entrypoint**

```bash
cd frontend
python -m venv .venv
source .venv/bin/activate            # Windows: .venv\Scripts\activate
pip install -r requirements.txt uvicorn[standard]
python -m api.seed                   # optional тАФ reset & re-seed
uvicorn api.index:app --reload --host 127.0.0.1 --port 8000
```

**Option B: legacy `backend/` folder**

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python seed.py
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Either way, the API is served at `http://127.0.0.1:8000` with all routes under `/api/*`. Docs at `http://127.0.0.1:8000/docs`.

### Terminal 2 тАФ Frontend

```bash
cd frontend
bun install
bun run dev
```

`frontend/.env.local` already points `NEXT_PUBLIC_API_URL` at `http://127.0.0.1:8000` so the browser hits the local uvicorn server during development. In production this env var is absent and the client falls back to same-origin `/api/*`.

App available at `http://localhost:3000`.

## Pages

| Path                            | Purpose |
|---------------------------------|---------|
| `/`                             | Redirects to `/wc/home` |
| `/wc/home`                      | Landing dashboard тАФ profile, action buttons, upcoming/recent meetings |
| `/wc/join`                      | Join by meeting ID + display name + optional passcode |
| `/meeting/schedule`             | Create a scheduled meeting (title, date/time, duration, passcode) |
| `/wc/{meetingId}/start`         | Meeting room тАФ video-grid layout, participants panel, host controls |

Query params on the meeting room:
- `host=1` тАФ the current user auto-joins as the host with their profile name
- `name=<display name>` тАФ pre-fill the display name (skip the join gate)
- `muted=0|1`, `video=0|1` тАФ initial mic/camera state
- `pwd=<passcode>` тАФ passcode

## API surface

All routes are mounted under `/api`.

| Method | Path | Purpose |
|--------|------|---------|
| GET    | `/api` | Health |
| GET    | `/api/me` | Current default user |
| GET    | `/api/meetings/upcoming` | Scheduled meetings in the future |
| GET    | `/api/meetings/recent`   | Ended meetings, most recent first |
| POST   | `/api/meetings/instant`  | Create + start an instant meeting |
| POST   | `/api/meetings/schedule` | Schedule a meeting (returns invite/join URLs) |
| GET    | `/api/meetings/{id}`     | Meeting details |
| POST   | `/api/meetings/{id}/validate` | Verify passcode for join |
| POST   | `/api/meetings/{id}/start` / `/end` | Change meeting status |
| GET    | `/api/meetings/{id}/participants` | Live participant list |
| POST   | `/api/meetings/{id}/participants` | Join meeting |
| PATCH  | `/api/meetings/{id}/participants/{pid}` | Toggle mute/video/hand |
| POST   | `/api/meetings/{id}/participants/{pid}/leave` | Leave meeting |
| POST   | `/api/meetings/{id}/participants/{pid}/remove` | Host removes participant |
| POST   | `/api/meetings/{id}/participants/mute-all` | Host mutes all non-host participants |

## Database schema

Three tables (see `frontend/api/models.py`):

- **user** тАФ `id, name, email, avatar_color, created_at`
- **meeting** тАФ `id, meeting_id (unique), host_id, title, description, passcode, scheduled_start, duration_minutes, status, is_instant, started_at, ended_at`
- **participant** тАФ `id, meeting_id, display_name, user_id?, is_host, is_muted, is_video_on, is_hand_raised, joined_at, left_at, removed`

The first person to join a meeting becomes the host if there isn't one already; when joining via the "New Meeting" shortcut (`?host=1`), the default user's name is used and they are marked host on their participant row.

## Assumptions

- **No authentication.** The backend has a single default user (`Alex Chen`) served from `GET /api/me`. Any request the app makes acts as this user.
- **No real audio/video streaming.** Tiles render an avatar with the participant's initials; mic/camera toggles only flip a database flag.
- **State polling.** The meeting room polls `/participants` every 2.5s instead of using WebSockets. Sufficient for a demo / RL gym environment.
- **Per-tab identity in the meeting room.** The joined participant ID is stored in `sessionStorage`, so opening a meeting URL in a new tab lets the new tab join as a distinct participant. Great for testing multi-user flows locally.
- **Timezone handling.** SQLite stores naive UTC datetimes; the frontend appends `Z` before parsing so the meeting timer and scheduled times are correct regardless of the browser timezone.
- **Ephemeral SQLite on Vercel.** State is reset on cold starts because the DB lives in `/tmp`. This matches the RL-gym use case; swap the DB URL to move to durable storage.
- **Passcodes** are optional. Scheduled meetings auto-generate a passcode when the field is left blank; instant meetings never require one.

## Development notes

- Next.js 16 makes `params` and `searchParams` promises тАФ all pages `await` them.
- Turbopack is on by default in both `next dev` and `next build`.
- The React Compiler is enabled (`reactCompiler: true` in `next.config.ts`).
- Tailwind v4 is used with the CSS `@theme` directive (see `src/app/globals.css`) instead of a `tailwind.config.js`.
- `frontend/api/index.py` prefixes every route with `/api` so it works identically behind the Vercel rewrite and under a local uvicorn server.
- FastAPI's `lifespan` handler calls `seed_if_empty()` тАФ safe to invoke on every cold start; it inserts sample rows only when the DB has no users.
